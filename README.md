An append-only log of decisions, rules, and constraints that agents
write and inherit across context resets.
Most harness memory treats the agent as the protagonist, building a
generalized user-profile of preferences. Priors is an experiment where
the project's trajectory is primary. Transient agents simply read
from and append to a typed log of decisions, constraints, rules, and
dead-ends. It's built off the idea that harness memory belongs to
the project, not an agent.
